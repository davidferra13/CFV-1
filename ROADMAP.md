## PRIORITY 1: Menu Builder System (THE CORE)

This is the single most important feature in ChefFlow. No tool like this exists anywhere. Build it right.

### What It Must Do

A chef needs to build menus by pulling from their recipe library, see real-time food cost as they build, and have the system automatically generate shopping lists, prep lists, and cost breakdowns.

### The Chain (all connected, all live-updating)

```
Menu → Courses → Dishes → Recipes → Components → Ingredients (with portions, scaled by guest count)
```

Every level is connected. Change the guest count, everything recalculates. Swap a dish, the cost updates. Add a recipe, the shopping list grows.

### Tasks

- [x] **Recipe Library** - Chef's personal recipe database with real portion sizes, yields, ingredients with costs, prep time, cook time, equipment needed. Recipes are entered manually by the chef (NEVER generated by AI). Each recipe has components (sub-recipes like sauces, stocks) and ingredients with exact quantities per portion. _(lib/recipes/actions.ts, 1500+ lines, full CRUD + sub-recipes + cost summaries)_

- [x] **Ingredient Database** - Master list of ingredients with current prices (manually entered by chef, updated over time). Unit conversions (oz to lb, ml to cups, etc.). Seasonal availability flags. Supplier/vendor tracking. _(lib/ingredients/, vendor_items table, recipe_ingredients with units)_

- [x] **Menu Builder UI** - Click-to-add interface. Chef selects from their recipe library to build a menu. Organized by course (appetizer, main, dessert, etc.). Shows running food cost percentage as dishes are added. Guest count input that scales everything automatically. _(components/menus/menu-doc-editor.tsx 1400+ lines, recipe-link-picker.tsx, culinary/MenuEditor.tsx 890 lines. Recipe linking + live costing added 2026-03-13.)_

- [x] **Menu Costing Engine** - Real-time food cost calculation per dish, per course, per menu. Ingredient cost rolled up from recipe portions scaled to guest count. Target food cost % with visual indicator (green/yellow/red). _(DB functions: compute_recipe_cost_cents, compute_menu_cost_cents. Views: recipe_cost_summary, menu_cost_summary, dish_component_summary. Live sidebar in editor added 2026-03-13.)_

- [x] **Shopping List Generator** - Auto-generated from finalized menu + guest count. Consolidated across all recipes (no duplicate ingredients). Quantities scaled to portions. Organized by store section or vendor. _(lib/culinary/shopping-list-actions.ts 343 lines + lib/formulas/grocery-consolidation.ts 781 lines. Deterministic, no AI. Components/culinary/ShoppingListGenerator.tsx 268 lines.)_

- [x] **Prep List Generator** - Auto-generated from menu recipes. Timeline working backwards from service time. What can be done days ahead vs. day-of vs. last-minute. _(lib/menus/actions.ts getMenuPrepTimeline, components/menus/prep-timeline-view.tsx. Components have prep_day_offset, prep_time_of_day, is_make_ahead fields.)_

- [x] **Menu Templates** - Save menus as templates for reuse. Clone and modify for similar events. Track which menus were used for which clients (never repeat for returning clients unless requested). _(is_template flag on menus, duplicateMenu action, previous menus shown in editor sidebar.)_

## PRIORITY 2: Inquiry Pipeline (Raw Feed to Refined Dashboard)

### What It Must Do

Every inquiry from every source funnels into one place. The chef sees a raw real-time feed, plus a refined dashboard for managing active leads.

### Tasks

- [x] **Raw Inquiry Feed** - Real-time feed of every incoming inquiry, unfiltered. Shows source (website, email, Instagram, text, referral, marketplace). Timestamp, basic info preview. _(Inquiry pipeline exists with source tracking, GOLDMINE lead scoring, real-time dashboard.)_

- [x] **Refined Inquiry Dashboard** - Scored and organized view. Lead score (already built via GOLDMINE), follow-up status, response status. Filter by source, date, score, status. Quick-action buttons (respond, archive, convert to event). _(Full dashboard with filters, status tabs, quick actions. GOLDMINE scoring integrated.)_

- [x] **Client History on Inquiry** - When an inquiry comes from a returning client, instantly surface: past events, menus served, dietary notes, preferences, last interaction date. _(Client detail pages show event history, menu history, dietary data. Editor sidebar shows previous menus for returning clients.)_

- [x] **Multi-Source Intake** - Website embed (already built). Email (GOLDMINE already extracts). Manual entry form for phone/text inquiries. _(Website embed, email parsing, manual entry all exist. Marketplace parsers built for multiple platforms.)_

## PRIORITY 3: Time Tracking

### What It Must Do

Track every hour the chef works, automatically categorized. Prep, shopping, travel, service, cleanup, admin, consultation. Show the chef their real hourly rate per event.

### Tasks

- [x] **Time Entry** - Simple start/stop timer or manual entry. Categories: prep, shopping, travel, service, cleanup, admin, consultation, menu development, tasting. Attach to a specific event or mark as general business. _(components/events/time-tracking.tsx 346 lines. 5 event activity types + admin time logging at /insights/time-analysis.)_

- [x] **Per-Event Time Summary** - Total hours by category for each event. Calculated hourly rate (revenue minus food cost, divided by hours). _(lib/intelligence/event-profitability.ts calculates effectiveHourlyRateCents. Shown on event detail money tab.)_

- [x] **Business-Wide Time Dashboard** - Weekly/monthly hours by category. Average hourly rate across all events. _(Admin time analysis at /insights/time-analysis with category breakdown + progress bars. Event profitability aggregation in intelligence hub.)_

## PRIORITY 4: Quote Builder (Connected to Menu Costing)

### What It Must Do

### Tasks

- [x] **Quote Composition** - Full quote form with pricing models, deposit settings, per-person pricing, guest count tiers. Menu food cost hint card shows live food cost from linked menu (added 2026-03-13). PricingSuggestionPanel and SmartPricingHint provide data-driven guidance. _(components/quotes/quote-form.tsx, lib/quotes/actions.ts, lib/pricing/compute.ts)_

- [x] **Quote vs. Actual Reconciliation** - After event: compare quoted costs to actual spend. Where did the chef go over? Where was there margin? _(Event detail money tab shows: revenue vs expenses, gross profit, food cost estimated vs actual with delta %, budget tracker (quoted vs actual), effective hourly rate. Intelligence hub aggregates by occasion and guest bracket. Smart pricing hints feed back into future quotes.)_

- [x] **Quote Templates** - Save common quote structures. Quote versioning and history. _(components/quotes/quote-version-history.tsx, durable drafts system.)_
