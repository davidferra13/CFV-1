## 1. CHEF WEBSITE BUILDERS

### 1A. Private Chef Manager (privatechefmanager.com)

**How it works:**

- Provides a professional web address in the format ChefYourName.com
- All-in-one platform: website + booking + scheduling + client management + quoting
- Clients book directly through the chef's website (no back-and-forth emails)
- System handles availability, client requests, scheduling, and custom quotes

**Plans & Pricing:**

- Three tiers: Solo chef, Scaling chef (website + analytics + support), Agency/team (multi-chef)
- Flat 2.9% service fee only when booked (no monthly subscription fee visible)
- Specific dollar amounts not publicly listed; requires signup/contact

**Customization:**

- Showcase dishes, share culinary journey, personality-driven branding
- Integrated booking system eliminates email scheduling
- Multi-language support (English, French, Spanish, Italian, Portuguese)

**Assessment:** Direct competitor to ChefFlow's potential website builder. Their ChefYourName.com approach is the model to beat. The 2.9% transaction fee is competitive but locks chefs into their payment processing.

**Must-have pages/sections:**

---

### 1H. Minimum Viable Chef Website That Converts

Based on research, the minimum viable site needs just 5 elements:

1. **One beautiful hero image** of food
2. **A clear service description** (what you do, where you do it)
3. **Social proof** (even 2-3 testimonials)
4. **A booking/inquiry form** (name, date, guests, dietary needs, phone/email)
5. **Contact info visible on every page** (phone number, service area)

A one-page site with these 5 elements will outperform a 10-page site that buries the inquiry form. The conversion killer is friction: if a client can't figure out how to contact the chef within 10 seconds, they leave.

## 2. CUSTOM DOMAINS

### 2A. Technical Approaches

**CNAME Records (Standard):**

- User adds CNAME record pointing their subdomain (e.g., app.chefname.com) to your platform
- Simple, well-understood, works for subdomains
- Cannot be used for root/apex domains (chefname.com) per DNS spec

**A Records (Root Domains):**

- User adds A record pointing root domain to platform IP
- Required for apex domains (chefname.com without www)
- Requires stable IP address

**Cloudflare for SaaS (Best for Scale):**

- Purpose-built for multi-tenant custom domains
- $2/month per custom hostname (some sources say $0.10/domain)
- 100 free custom hostnames included
- Automatic SSL via domain validation (TXT, HTTP, or CNAME)
- Full TLS lifecycle management (issuance + renewal)
- Custom metadata per hostname (via Workers for per-tenant behavior)
- Supports 20,000+ domains under one account

**Vercel Custom Domains (Our Stack):**

- Domains API for programmatic add/remove
- SDK: `projectsAddProjectDomain()` to add, verification functions
- Automatic SSL certificate issuance
- Anycast network for low-latency CDN
- Real-world: Super.so serves 15,000+ custom domains from one codebase
- Domain purchase at registrar cost (up to 50% savings on popular TLDs)
- Multi-tenant architecture: middleware reads hostname, routes to tenant

**Netlify Custom Domains:**

- Automatic Let's Encrypt SSL on domain add
- Wildcard certificates for Netlify-managed DNS
- Pro plan required for wildcard subdomains
- Good for static sites, less ideal for dynamic Next.js apps

---

### 2B. How Platforms Handle Domain Provisioning

**Shopify:**

- Settings > Domains > Connect existing domain
- A record: 23.227.38.65 (root domain)
- CNAME: www -> shops.myshopify.com
- Auto-connect for Google Domains and IONOS (just login)
- SSL auto-provisioned after correct DNS
- Propagation: 1-4 hours typical, up to 48 hours

**Carrd:**

- Enter domain in site settings
- A record: @ -> target IP
- CNAME: www -> bare domain
- Auto-initialization (usually under 1 hour)
- SSL included automatically
- Plans: $9/year (Pro Lite), $19/year (Pro Standard), $49/year (Pro Plus)

**Common UX Pattern Across Platforms:**

1. User enters desired domain in dashboard
2. Platform shows exact DNS records to add (with copy buttons)
3. User adds records at their registrar
4. Platform polls for DNS propagation
5. Shows "Pending" -> "Connected" status
6. SSL auto-provisions once DNS resolves

---

### 2C. SSL Certificate Automation

**Let's Encrypt (Free):**

- Free DV certificates via ACME protocol
- 90-day validity, auto-renewable
- Wildcard support via DNS-01 challenge (ACME v2)
- Rate limits: 50 certificates per registered domain per week
- Recommended client: Certbot
- One DV cert per tenant domain is the most scalable pattern
- SNI (Server Name Indication) selects correct cert per hostname

**Cloudflare SSL:**

- Included with Cloudflare for SaaS
- Automatic issuance and renewal
- Advanced Certificate Manager: $10/month per domain
- Custom certificates: Business ($200/mo) or Enterprise

**Platform-Managed (Vercel/Netlify):**

- Zero config: add domain, SSL auto-provisions
- No per-certificate cost
- Handled entirely by the platform

### 2E. DNS Propagation UX Best Practices

**What top platforms do:**

1. **Show exact records to add** with copy-to-clipboard buttons
2. **Provider-specific instructions** (GoDaddy, Namecheap, Cloudflare, Google Domains)
3. **Real-time verification** with "Check DNS" button
4. **Clear status states:** Pending -> Verifying -> Connected (or Error with diagnosis)
5. **"We see this record / we expected that record" diff** (dramatically reduces support tickets)
6. **Email notification** when domain is fully connected
7. **Troubleshooting guide** for common issues (wrong record type, TTL too high, proxy interfering)

**Key UX insight:** Support tickets drop dramatically once the DNS onboarding screen includes a simple diff showing what was found vs. what was expected.

## 3. WHITELABELING

### 3A. Dubsado's Whitelabeling (Gold Standard for Service Businesses)

**What can be customized:**

- Custom URL: portal.yourdomain.com instead of hello.dubsado.com
- Logos on forms, contracts, proposals, invoices
- Brand colors (accent color on all buttons, emails, scheduler)
- Custom fonts
- Favicon and window name
- Facebook/social preview text and image
- Custom CSS for advanced styling beyond built-in options
- Email sender address

**What gets removed:**

- Zero Dubsado branding visible to clients
- No "powered by Dubsado" anywhere
- Complete brand invisibility: clients only see the chef's brand

**How it works technically:**

- White label URL via CNAME record (portal.yourdomain.com -> dubsado)
- Brand color applies to all client-facing elements
- Custom CSS injection for advanced theming
- Email sent from user's connected email address, not Dubsado's

**Assessment:** This is the benchmark for ChefFlow. Dubsado proves that service professionals (photographers, event planners, and by extension chefs) deeply value complete brand ownership of client-facing touchpoints.

### 3C. Technical Implementation

**CSS Theming:**

- Store per-tenant theme config (primary color, accent color, font family, logo URL)
- CSS custom properties (variables) applied at runtime
- Example: `--brand-primary: #chef-chosen-color`
- Some platforms allow full custom CSS injection for advanced users
- Tailwind: can be configured with dynamic theme values per tenant

**Logo Replacement:**

- Store logo URL per tenant in database
- Replace platform logo with tenant logo in all client-facing pages
- Include in emails, PDFs, invoices, contracts
- Need multiple formats: square (favicon), horizontal (header), stacked

**Custom Email Sender:**

- Connect tenant's email via SMTP or OAuth (Gmail, Outlook)
- SPF/DKIM/DMARC records for deliverability
- "From" address shows tenant's email, not platform's
- Reply-to goes directly to tenant
- Dubsado model: email communications completely white labeled from chosen address

**Branded Login Pages:**

- Custom login URL (app.chefname.com/login)
- Tenant logo, colors, background image on login screen
- No platform branding visible
- Session cookies scoped to tenant's domain

---

### 3D. What Chefs Actually Want from Whitelabeling

Based on research across restaurant/chef software platforms and Dubsado user feedback:

**Tier 1 (Must-Have):**

- Remove "Powered by ChefFlow" badge from client-facing pages
- Chef's logo on invoices, contracts, proposals, emails
- Chef's brand colors on client portal and booking pages
- Emails sent from chef's email address (not noreply@chefflow.com)

**Tier 2 (High Value):**

- Custom domain for client portal (portal.chefname.com)
- Chef's branding on the inquiry/booking widget
- Custom favicon

**Tier 3 (Nice-to-Have):**

- Custom fonts
- Custom CSS injection
- Branded mobile app icon (PWA manifest customization)
- Custom 404/error pages

**Key insight from restaurant white-label research:** Chefs/restaurants want "full control over branding, customer data, and promotions, and no hefty commission fees." The biggest frustration with white-label solutions is when they're "generic apps that other restaurants may use as well" with only superficial branding changes. True white-labeling means the client never knows what platform is behind the scenes.

### 4C. Schema.org Structured Data for Chefs

**Applicable schema types:**

```json
{
  "@context": "https://schema.org",
  "@type": "FoodService",
  "name": "Chef [Name]",
  "description": "Private chef services in [City]",
  "areaServed": {
    "@type": "City",
    "name": "[City, State]"
  },
  "servesCuisine": ["Italian", "French", "Mediterranean"],
  "telephone": "+1-555-123-4567",
  "url": "https://chefname.com",
  "image": "https://chefname.com/hero.jpg",
  "priceRange": "$$$$",
  "aggregateRating": {
    "@type": "AggregateRating",
    "ratingValue": "4.9",
    "reviewCount": "47"
  },
  "hasOfferCatalog": {
    "@type": "OfferCatalog",
    "name": "Private Chef Services",
    "itemListElement": [
      {
        "@type": "Offer",
        "itemOffered": {
          "@type": "Service",
          "name": "Private Dinner Party",
          "description": "Multi-course dinner for up to 12 guests"
        }
      }
    ]
  }
}
```

**Most relevant schema types:**

- `FoodService` (under LocalBusiness)
- `FoodEstablishment` (alternative)
- `ProfessionalService` (for personal chefs as service providers)
- `Review` / `AggregateRating` (social proof in search results)
- `Service` / `Offer` (for service listings)
- `Person` (for chef bio with `jobTitle: "Private Chef"`)
- `Menu` (for sample menus)

**Implementation:** JSON-LD is Google's recommended format. Should be auto-generated from chef's profile data (name, location, services, reviews).

### 4D. Local SEO Strategy

**Google Business Profile:**

- Google shut down free GBP websites (March 2024), pushing users to third-party builders
- GBP remains essential for local pack results and Google Maps
- API access requires verification as a large business or third-party platform
- For SaaS integration: guide chefs through GBP setup, don't try to automate it

**What works for local chef SEO:**

1. Consistent NAP (Name, Address, Phone) everywhere
2. City/neighborhood mentions in page content
3. Service area pages (one page per city served)
4. Google Business Profile with regular posts
5. Local directory listings (Yelp, Thumbtack, etc.)
6. Reviews on Google, Yelp, and on the chef's own site
7. Location-based schema markup

**Key stat:** 4 out of 5 consumers use search engines for local searches.

## 5. STRATEGIC IMPLICATIONS FOR CHEFFLOW

### 5A. Website Builder Opportunity

**Market gap:** No existing platform combines chef-specific website building with full business ops (like ChefFlow already has). Private Chef Manager comes closest but is a separate platform from what chefs use day-to-day.

**ChefFlow advantage:** We already have the chef's data (services, menus, recipes, testimonials, location, dietary capabilities). A website builder that auto-populates from existing ChefFlow data would be zero-effort for the chef.

**Competitive positioning:**

- Squarespace/Wix: Beautiful but generic, no chef ops integration
- Cast Iron: Food-specific but targets bakers/cottage food, 10% take rate
- Private Chef Manager: Chef-specific but siloed platform
- ChefFlow: Chef-specific + ops integrated + auto-populated from existing data

### 5B. Custom Domain Approach

**Recommended: Vercel's Domains API (we're already on Vercel)**

- $0 additional cost on current plan
- Programmatic domain add/remove via API
- Auto SSL
- Middleware-based tenant routing (already have multi-tenant architecture)
- Proven at scale (15,000+ domains on single codebase)

**Fallback: Cloudflare for SaaS**

- If Vercel limits custom domains on our plan tier
- $0.10-$2/domain/month
- More control over DNS and SSL configuration

**Subdomain approach (Phase 1):**

- chef-name.cheflowhq.com (free, instant, no DNS setup for chefs)
- Wildcard DNS + SSL on our domain
- Zero friction for chefs, immediate value

**Custom domain (Phase 2):**

- chefname.com or book.chefname.com
- Chef adds CNAME/A record at their registrar
- ChefFlow provides step-by-step DNS guide with verification
- Pro tier feature

### 5C. Whitelabeling Tier Strategy

**Free Tier:**

- Chef's name and logo displayed prominently
- Small "Powered by ChefFlow" footer link
- ChefFlow subdomain (chef.cheflowhq.com)

**Pro Tier:**

- Remove "Powered by ChefFlow" badge
- Custom brand colors on client-facing pages
- Custom logo on invoices, contracts, emails
- Custom domain support

**Enterprise/Agency:**

- Custom email sender domain
- Custom CSS injection
- Branded login page
- PWA manifest customization
- Complete brand invisibility

### 5D. SEO as a Competitive Moat

Auto-generated schema markup from ChefFlow data is a massive differentiator. No generic website builder does this. If every ChefFlow chef site automatically has:

- LocalBusiness/FoodService schema
- Service catalog schema
- Review/rating schema
- Proper meta tags with city + service keywords

...then ChefFlow sites will systematically outrank Squarespace/Wix chef sites in local search. This is a feature that sells itself.
