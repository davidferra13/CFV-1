## What Exists

### Service Worker (`public/sw.js`)

Generated by `@ducanh2912/next-pwa` during build. Uses Workbox with:

- **Precaching**: All Next.js static chunks, CSS, fonts, and public assets
- **Offline fallback**: `/offline.html` served when navigation requests fail
- **Caching strategies**:
  - `CacheFirst` for static assets (JS bundles, fonts, images)
  - `NetworkFirst` for pages, API routes, and Next.js RSC payloads
  - `StaleWhileRevalidate` for CSS, fonts, and Next.js image optimization
  - Cross-origin requests: `NetworkFirst` with 10-second timeout

### Manifest (`public/manifest.json`)

- `name`: ChefFlow
- `short_name`: ChefFlow
- `display`: standalone (hides browser chrome when installed)
- `start_url`: /dashboard
- `theme_color`: #e88f47 (brand orange)
- `background_color`: #faf9f7 (warm stone)
- Shortcuts: New Event (`/events/new`), Inbox (`/inbox`)
- Four icon sizes: 192px, 512px, maskable 192px, maskable 512px (PNG + SVG)

### Next.js Config (`next.config.js`)

```js
withPWA = require('@ducanh2912/next-pwa').default({
  dest: 'public', // output sw.js here
  register: true, // auto-register SW in the app
  skipWaiting: true, // activate new SW immediately
  disable: process.env.NODE_ENV === 'development', // off in dev
})
```

### App Layout (`app/layout.tsx`)

- `metadata.manifest: '/manifest.json'` — links the PWA manifest
- `metadata.appleWebApp: { capable: true, statusBarStyle: 'black-translucent' }` — iOS home screen support
- `viewport.themeColor: '#e88f47'` — matches manifest

## Dev Note

The service worker is **disabled on localhost** (`NODE_ENV === 'development'`). To test PWA behavior:

1. Build: `npm run build`
2. Start: `npm start`
3. Open Chrome DevTools → Application → Service Workers

Or deploy to staging and use Lighthouse PWA audit.
