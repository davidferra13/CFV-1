# Storyboard #7: Event Handoff: Chef to Chef (Dual-Perspective)

> "Can't make the dinner? Hand it off in 30 seconds."
> Duration: 75 seconds | Format: 1920x1080 (YouTube 16:9) | Auth: chef A + chef B (dual context)

---

## The pitch in one sentence

Life happens. A chef gets sick, has a conflict, or gets a better opportunity. Instead of scrambling to find a replacement via text and losing all the event context, ChefFlow lets them hand off the entire event (client history, dietary needs, menu, timeline) to another chef in 30 seconds. The receiving chef walks in fully prepared.

## PRODUCTION NOTES

- **Triple context required.** Three Playwright browser contexts: Chef A, Chef B, Client. Or record Chef A + Chef B as dual context, then add the client notification scene separately.
- The handoff feature must be functional end-to-end. If it's not fully built, simulate: pre-set the event ownership change and record each perspective seeing the "after" state.
- The dietary matrix with per-guest allergies is critical to show. Seed the event with 3+ guests with different dietary needs.
- Chef B's client profile view should show past event history with Chef A. This proves the entire client record transfers, not just the single event.
- The client notification scene (0:52-0:60) can be recorded separately from the dual-chef recording. It's a quick scene showing either an email or the client portal.
- If handoff notifications aren't built yet, show a simulated notification toast on Chef A's and Chef B's screens.
