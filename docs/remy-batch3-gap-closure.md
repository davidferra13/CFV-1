# Remy Batch 3: Full Gap Closure

**Date:** 2026-03-06
**Branch:** `feature/risk-gap-closure`

## What Changed

Every app feature that existed without Remy integration now has it. Before this change, Remy covered ~85 intelligence queries. After: ~120+.

## New Capabilities (35 commands)

### Hub Circles (3 commands)

- `circles.list` - List all circles with member count, unread, activity
- `circles.unread` - Total unread messages across circles
- `circles.events` - Events linked to a specific circle

### Rate Card (1 command)

- `rate_card.summary` - Default rate, service types, avg per-head from recent events

### Tasks / Kanban (3 commands)

- `tasks.list` - Task board items with optional status filter
- `tasks.by_date` - Tasks due on a specific date
- `tasks.overdue` - Past-due uncompleted tasks

### Travel / Logistics (2 commands)

- `travel.plan` - Travel plan for a specific event (legs, times, ingredients)
- `travel.upcoming` - All upcoming travel legs

### Commerce / POS (5 commands)

- `commerce.products` - Active product inventory with pricing
- `commerce.recent_sales` - Recent POS transactions
- `commerce.daily_report` - Today's sales report
- `commerce.product_report` - 30-day product performance
- `commerce.inventory_low` - Low stock alerts

### Daily Ops (2 commands)

- `daily.plan` - Today's daily operations plan
- `daily.stats` - Daily plan completion statistics

### Priority Queue (1 command)

- `queue.status` - Priority queue ranked by urgency, with urgent/high/normal breakdown

### Stations / Kitchen Ops (4 commands)

- `stations.list` - Kitchen stations with items and components
- `stations.detail` - Full station detail
- `stations.ops_log` - Kitchen operations log (optionally filtered by station)
- `stations.waste_log` - 30-day food waste summary

### Testimonials (2 commands)

- `testimonials.list` - All testimonials with ratings, averages, approval status
- `testimonials.pending` - Testimonials awaiting approval

### Partners / Referrals (3 commands)

- `partners.list` - Referral partner directory
- `partners.events` - Events generated by a specific partner
- `partners.performance` - Partners ranked by events generated

### Activity Feed (2 commands)

- `activity.feed` - Recent platform activity (20 most recent events)
- `activity.engagement` - Client engagement statistics

### After-Action Reviews (4 commands)

- `aar.list` - Recent AARs with ratings
- `aar.stats` - Aggregate AAR statistics
- `aar.events_without` - Completed events missing an AAR
- `aar.forgotten_items` - Most frequently forgotten items from AAR data

### Waitlist (1 command)

- `waitlist.status` - Clients currently on the waitlist

### Navigation Routes (+15 routes)

Added to the "go to [page]" command: circles, hub, rate card, rates, tasks, travel, commerce, pos, register, daily ops, daily, queue, stations, testimonials, partners, activity, consulting, aar, waitlist.

## Architecture

All new commands follow the established pattern:

1. **Deterministic patterns** in `command-intent-parser.ts` bypass Ollama for common phrases
2. **Task descriptions** in `command-task-descriptions.ts` feed the Ollama system prompt for freeform queries
3. **Case handlers** in `command-orchestrator.ts` route task types to executor functions
4. **Executor functions** in `remy-intelligence-actions-3.ts` query the database via existing server actions

Everything is Formula > AI. Zero Ollama dependency for any of these 35 commands. All tenant-scoped via `requireChef()`.

## What's Still Intentionally Excluded

- **Prospecting** - admin-only, by design
- **Consulting** - static content page, no data to query
- **Culinary Board** - community/social feed, not enough structured data for intelligence queries
- **Commerce write operations** (refunds, POS transactions) - restricted for audit safety
- **Task completion via Remy** - tasks need real-time execution, not draft-approval flow
