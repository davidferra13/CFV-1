## A-HA MOMENTS (4 total)

1. **Intelligence:** Dietary matrix shows every guest's restrictions in one visual grid. No spreadsheet, no sticky notes.
2. **Speed:** Menu compliance badges check every dish against every guest's restrictions automatically.
3. **Both sides:** Group members each see their dietary accommodations highlighted and approve individually.
4. **Scale:** After approval, packing list and prep schedule auto-generate. The system does the busywork.

## CAPTION COUNT: 11 total

## PRODUCTION NOTES

- This video is entirely from the chef's perspective (except the brief approval montage).
- The dietary matrix grid is the star visual. If it's not built as a dedicated component, create a table view showing guests vs. restrictions. This is worth building even just for the video.
- Menu compliance badges (checkmarks showing which restrictions each dish satisfies) may not be built. If not, show the dietary restrictions next to each menu item as text tags (e.g., "(V)(NF)(GF)").
- The approval montage (Scene 5, 0:45-0:51) can be simulated: show pre-set approval states rather than real-time approvals from guest accounts.
- Packing list auto-generation from menu items is a powerful detail. If it's not automatic, show a manually populated packing list that looks like it was generated.
- Seed the event with 6 guests with diverse dietary needs: 2 vegan, 2 nut-free, 1 gluten-free, 1 no restrictions.
