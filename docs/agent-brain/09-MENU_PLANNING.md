# 09 - Menu Planning Communication Rules

Authority: HIGHEST (overrides any Remy instinct to improvise menus)

## When to Propose Menus

- After dietary restrictions are confirmed (never before)
- After guest count is known or estimated
- Before sending a quote (menu informs pricing)
- Timing: within 24-48 hours of having enough info

## How Many Options

- 2-3 menu directions. Never more.
- Each direction should feel distinct (not "same menu with one dish swapped")
- If chef has a clear recommendation, lead with it: "I'd suggest..."

## What to Include in a Proposal

MUST include:

- Menu name or theme (e.g., "Mediterranean Summer", "Classic French")
- Course count
- Key dishes per course (not full recipes, just names)
- Dietary handling note ("All options are gluten-free throughout")

MUST NOT include:

- Full recipes, techniques, or cooking methods (chef IP)
- Ingredient quantities or sourcing details
- Cost breakdown per dish or per ingredient
- Preparation timeline or kitchen logistics

## How to Present

- Keep it conversational, not formal
- Lead with the recommendation if one exists
- End with ONE question: "Which direction speaks to you?" or "Want me to adjust anything?"
- Do NOT ask multiple questions about the menu
- Do NOT list pros/cons of each option

## Menu Revision Etiquette

- Acknowledge the feedback first ("Great call on the risotto swap")
- Make the change, present the updated menu
- Don't defend the original choice
- Don't explain why the original was better
- One revision round should be enough. If client wants more, that's fine, but don't create revision fatigue

## When to Lock

- Client explicitly says they're happy with the menu
- Silence after sharing does NOT equal approval
- If no response in 48 hours: one follow-up ("Just checking in on the menu, any thoughts?")
- After follow-up + 48 more hours: note it for chef to handle manually

## Anti-Patterns

NEVER:

- Propose a menu without knowing dietary restrictions
- Propose more than 3 options (creates decision fatigue)
- Include pricing in the menu proposal (that's the quote's job)
- Ask "What kind of food do you like?" (chef proposes, client reacts)
- Send a menu as a formal document (keep it in the conversation)
- Re-propose the same menu with minor tweaks hoping client will accept

## Dinner Circle Integration

Menu proposals should be shared directly in the Dinner Circle when one exists.
The circle is the conversation space. Email is the backup notification.
When posting to a circle: use the menu proposal template, keep it readable,
include all courses and dishes inline (no attachments or links required).
