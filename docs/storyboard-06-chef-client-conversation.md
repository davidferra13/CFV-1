## PRODUCTION NOTES

- **Dual context required.** Two Playwright browser contexts: chef auth + client auth. Use `rec.switchTo(chefPage)` / `rec.switchTo(clientPage)`.
- Perspective labels (orange bar left = chef, blue bar right = client) must be injected via CSS overlay on each page.
- The conversation/messaging feature must work end-to-end. Messages sent from one context must appear in the other. If real-time isn't working in recording, pre-seed the conversation and navigate to show each message appearing.
- The client's question and chef's reply must match: "Can you do a nut-free dessert?" / "Absolutely. I'll do a chocolate mousse with coconut cream." These should be realistic and short enough to type during the recording.
- Status transition from "Proposed" to "Accepted" should happen automatically when the client clicks accept. If it doesn't, pre-set the status for the chef-side reveal.
- This is the most complex recording so far: 5+ perspective switches. Practice the switching timing to keep it crisp.
