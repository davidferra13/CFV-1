# User Feedback Log

> Track all feedback from real users (testers, beta users, paying customers).
> Every piece of feedback gets logged here — positive, negative, feature requests, bugs, confusion points.

## How to Log Feedback

Add entries in reverse chronological order (newest first). Every entry needs:

- **Date** — when the feedback was received
- **Source** — who said it (anonymize if needed: "Beta Tester A", "Client B")
- **Type** — Bug | UX Issue | Feature Request | Positive | Confusion
- **Description** — what they said (quote if possible)
- **Action** — what we did about it (or "Backlog" if deferred)
