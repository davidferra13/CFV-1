# SOP Version Control Policy

## Purpose

Ensure all SOP changes are documented, approved, and communicated to the team.

## Version Format

Use semantic format:

- `major.minor.patch`
- Example: `2.3.1`

## Change Types

- Major: policy or compliance-impacting changes
- Minor: process improvements without policy impact
- Patch: wording fixes or clarifications

## Required Process

1. Draft change and rationale.
2. Add entry to SOP change log.
3. Assign effective date.
4. Confirm training impact and retraining plan.
5. Communicate change to all relevant team members.
