## DC: Data Continuity (Does peak window data survive every journey?)

**DC1. When Chef A shares a recipe with Chef B, do all 6 peak window fields survive the copy?**
Chef A painstakingly sets peak_hours_min=24, peak_hours_max=48, safety_hours_max=72, storage_method=fridge, freezable=true, frozen_extends_hours=168 on her bolognese. She shares it with Chef B via the collaboration system. Chef B opens the recipe. Does he see the peak window data, or did `deepCopyRecipe` silently drop it?

**DC2. When a recipe is duplicated within the same chef's account, do peak fields carry over?**

**DC3. When a recipe is imported via CSV, can peak window columns be included?**

**DC4. When a recipe is imported via AI text parsing, does the parser attempt to extract peak window hints?**
Chef pastes: "This sauce holds beautifully for 3-4 days refrigerated and freezes well for up to 3 months." Does the AI import extract peak_hours_max=96, storage_method=fridge, freezable=true, frozen_extends_hours=2160? Or does it ignore timing information entirely?

**DC5. Does demo/seed data include peak window values?**
A new chef signs up and sees demo recipes. Do those recipes showcase peak windows with realistic values, or are all peak fields null, making the feature invisible during onboarding?

## AI: Remy Integration (Can the AI reason about peak windows?)

**AI1. When a chef asks Remy "generate a prep timeline for the Johnson dinner," does Remy use the deterministic peak-window engine or the Ollama-based generator?**
This is the single biggest gap. The `prep.timeline` command routes to `generatePrepTimelineByName` (Ollama, ignores peak windows). Meanwhile, the actual peak-window-aware engine (`computePrepTimeline`) runs silently in the background for dashboard prompts. Which system does the chef's explicit request hit?

**AI2. Can a chef tell Remy "set the peak window on my bolognese to 48-72 hours" and have it actually update?**
No `agent.set_peak_window` action exists. The `updateRecipePeakWindow` server action is only callable from the edit form UI. Can Remy set peak windows conversationally, or must the chef navigate to the recipe edit page?

**AI3. Can a chef ask Remy "what's the peak window for my bolognese?" and get a correct answer?**
No `agent.query_peak_window` action exists. `hasExplicitPeakWindow` and `getCategoryDefault` are never imported in the AI layer. When the chef asks, does Remy know, or does it hallucinate?

**AI4. When Remy loads context for a specific recipe conversation, does it include peak window data?**
`loadRecipeEntity` (remy-context.ts:2326) queries 14 fields but no peak fields. If a chef asks "can I make this the day before?", Remy needs peak_hours_min/max to answer truthfully. Can it?

**AI5. Does the Remy system prompt explain what peak windows are?**
`route-prompt-utils.ts` shows `[shop by DATE]` derived from peak windows, but never explains the concept. Does Remy understand WHY that grocery date exists? Can it explain it to the chef? Or does it just parrot the date?

**AI6. When a chef asks "when should I start prepping for the Saturday dinner?", does the intent parser route it correctly?**
No deterministic pattern exists for this query. It falls to LLM classification. Does it hit `prep.timeline` (wrong system anyway)? `scheduling.prep_blocks`? Or does it misroute entirely?

**AI7. When Remy says "You have 3 recipes without peak windows for an upcoming event," does it tell the chef HOW to fix it?**
The untimed items nudge exists in prep-prompts. But does Remy proactively suggest setting peak windows on specific recipes? Does it offer to navigate to the edit page? Or just state the problem?

**AI8. When a chef links a menu to an event via Remy, does Remy check peak window coverage?**
`agent.link_menu_event` links the menu but doesn't check if recipes have peak windows. Does Remy say "3 of 5 recipes on this menu are using default peak windows, want to set explicit ones?" Or does it silently proceed?

**AI9. When a chef changes an event date via Remy, does Remy warn about prep timeline impact?**
Moving a dinner from Saturday to Thursday shifts all prep days. Does `agent.update_event` warn "this will move your grocery deadline to Tuesday and prep start to Wednesday"? Or does it change the date without any timeline context?

**AI10. When a chef asks Remy "show my prep blocks for the Johnson dinner" and none exist yet, does Remy suggest creating them?**
`executePrepBlocks` only fetches existing blocks. If the chef hasn't created any yet, does Remy call `autoSuggestEventBlocks` (peak-window-aware) to generate suggestions? Or return "no blocks found"?
