# Storyboard #2: Full Lifecycle - Inquiry to Invoice

> "From first message to final payment. One platform."
> Duration: 90 seconds | Format: 1920x1080 (YouTube 16:9) | Auth: chef account

## The pitch in one sentence

A single lead comes in, gets scored, turns into an event, gets a quote, a proposal, a payment, and an invoice. The chef never leaves ChefFlow. This is the "holy shit, this does everything" video.

## PRODUCTION NOTES

- This video follows ONE event through its entire lifecycle. Use a single demo event with good data: a private dinner for 8, $3,200 total, interesting dietary notes (one vegan, one nut allergy), wine pairing add-on.
- The event should already exist in the agent account's data with a quote, payment history, and invoice. If not, seed it before recording.
- The Kanban board scene needs at least 8-10 events spread across different columns to look full and active. Seed enough events in various states.
- Status transitions may need to be pre-set rather than triggered live (transitioning from Draft to Proposed requires sending a real proposal). Pre-stage the event at each state needed for each scene.
- The proposal preview scene depends on how the proposal UI renders. If it's a modal, keep the background visible. If it navigates to a new page, use a scene cut.
- Record on beta for fast page loads. The 9-scene structure with scene cuts means any compile delay will destroy the pacing.
- This is the longest video (90s). Pacing must be tight. If any scene runs long during recording, trim the pauses, not the content.
