# Storyboard #5: The Embeddable Widget

> "Add one line of code. Never miss a lead again."
> Duration: 45 seconds | Format: 1920x1080 (YouTube 16:9) | Auth: chef account

## The pitch in one sentence

Any chef with a website can paste one script tag and instantly get a professional inquiry form that feeds directly into their ChefFlow pipeline with lead scoring. From website visitor to scored lead in seconds, without ever logging into ChefFlow.

## PRODUCTION NOTES

- Need a demo chef website to show the widget on. Options: (a) create a simple static HTML page with a hero image and the widget embedded, (b) use an existing chef website template, (c) use the ChefFlow public chef profile page as the "website."
- Option (a) is cleanest: create `scripts/demo-chef-website.html` with basic styling, a food hero image, brief bio text, and the embedded widget script. This page doesn't need to be hosted; Playwright can load a local HTML file.
- The widget form submission needs to actually create an inquiry in the pipeline for the next scene to work. If recording on beta, the widget API endpoint must be functional.
- Alternative: record scenes 3 and 4 separately. Show the widget form being filled, then cut to the pipeline where a matching inquiry already exists (pre-seeded). This avoids needing real-time form-to-pipeline flow during recording.
- The widget button must be visible and styled with brand colors. Check that `public/embed/chefflow-widget.js` renders correctly in the recording viewport.
- This is a short video (45s, 5 scenes). Pacing is critical. Every second counts. No lingering on any screen.
