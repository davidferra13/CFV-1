# V1 Module Ownership Registry

Status: initial human-readable scaffold
Date: 2026-05-01
Scope: module and submodule ownership for V1 architecture review

This registry bridges backlog module taxonomy and code ownership. It is not yet complete file-by-file ownership truth. Use it to decide where work belongs before implementation.

## Status Values

| Status         | Meaning                                                                           |
| -------------- | --------------------------------------------------------------------------------- |
| `known`        | Owner, anchors, invariants, and proof surface are clear enough for scoped builds. |
| `partial`      | Owner is clear, but anchors or proof surface need confirmation before edits.      |
| `unknown`      | Not enough ownership evidence. Route to module review.                            |
| `conflict`     | Multiple owners appear plausible. Resolve before building.                        |
| `needs-review` | Candidate owner exists, but assignment needs architecture review.                 |
