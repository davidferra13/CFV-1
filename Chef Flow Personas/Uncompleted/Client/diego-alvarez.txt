Primary Failure: Pricing Latency
The biggest issue is delay. I change guest count or add a course and the UI hesitates, spins, or recalculates slowly. That’s a broken feedback loop. Pricing must feel instantaneous. If it doesn’t, I don’t trust the number or the system.

Structural Issue: Server-Dependent Pricing
If every pricing update requires a round-trip to the server, you’ve already lost. Core pricing logic should be computed client-side or cached intelligently. Waiting on network latency for basic math is unacceptable.

Structural Issue: Jittery UI States
Totals flicker, jump, or briefly show incorrect numbers before settling. That inconsistency makes the system feel unstable. Pricing should be stable and atomic - no intermediate wrong states.

Structural Issue: No Offline Resilience
If my connection drops, the system becomes unusable. I should still be able to configure and see pricing locally. Losing that capability breaks continuity.
