Primary Failure: Inaccessible Pricing
If pricing isn’t readable by assistive technology, it doesn’t exist. Hidden or visually dependent pricing is a failure.

Structural Issue: Poor Semantic Structure
If totals and breakdowns aren’t properly labeled, screen readers can’t interpret them. That breaks the experience.

Structural Issue: Interaction Barriers
If I can’t adjust inputs (guest count, courses) easily via keyboard or assistive tech, the system is unusable.

Structural Issue: Dynamic Updates Not Announced
Real-time pricing changes must be communicated to assistive tools. Silent updates create confusion.

I avoid: visually dependent interfaces
I evaluate tools by: accessibility compliance and usability

Pass / Fail Conditions
