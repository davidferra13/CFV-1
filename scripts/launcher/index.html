How a Page Load Works (The Full Chain)

1
Browser sends request
User types beta.cheflowhq.com/dashboard
2
Cloudflare DNS resolves the domain
beta.cheflowhq.com → Cloudflare tunnel
3
cloudflared daemon receives the request
Through the tunnel on the Raspberry Pi
4
Forwarded to localhost:3100 on Pi
cloudflared passes it to the Node.js server
5
Next.js middleware checks auth
Talks to Supabase: "Is this user valid?"
6
Server renders the page
Calls server actions → queries Supabase, Ollama, Stripe as needed
7
HTML + JavaScript sent back through tunnel
Response travels back the same path
8
Page loads — user can click things
Browser renders HTML, JavaScript hydrates, page is interactive

HTTP Status Codes (What Those Numbers Mean)

200
OK — everything worked
301
Moved — redirect
400
Bad request — sent wrong data
401
Unauthorized — need to log in
403
Forbidden — logged in but not allowed
404
Not found — doesn't exist
500
Server error — the server broke

Vocabulary Quick Reference

Term Plain English
Terminal The window that shows text. Like a TV screen — just a container.
Shell The program inside the terminal that reads your commands (Bash, PowerShell).
CLI A program you interact with by typing, not clicking. Opposite of GUI.
Process Any program currently running. Your browser, Ollama, VS Code — all processes.
Daemon A process that runs silently in the background with no window.
Service A daemon managed by the OS — it auto-starts on boot and auto-restarts on crash.
Port A numbered door on a computer. Each program listens on a specific port.
localhost Your own machine. localhost:3100 = "port 3100 on this computer."
SSH A way to open a terminal on a remote machine (like your Pi from your PC).
Tunnel Makes a local service reachable from the internet. Pi → Cloudflare → beta.cheflowhq.com.
API How programs talk to each other — structured requests and responses, no GUI.
Endpoint A specific URL in an API. /api/tags and /api/ps are endpoints.
GET / POST GET = "give me data" (read). POST = "here's new data" (create/update).
Log A record of what a program is doing. That scrolling text = a log.
Health Check A periodic "are you alive?" request. The /api/ps calls in Ollama's log.
Hot Reload Save a file → app updates instantly without restarting.
Build Converting human-readable code into optimized code the computer runs fast.
Deploy Moving built code to a server where users can access it.
Headless A computer with no monitor/keyboard. Your Pi is headless — controlled via SSH.
OOM Out of Memory — a process tried to use more RAM than available. OS kills something.
Latency How long a message takes to travel between two points. Lower = better.
Flag A modifier on a command: --oneline, -v, --force. The dashes give it away.
stdout / stderr Two output channels: normal output (stdout) and error messages (stderr).
PID Process ID — every running program gets a unique number. Used to kill stuck processes.
