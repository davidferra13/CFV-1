# Client Portal

**What:** Authenticated client access to their events. Where clients see their bookings, RSVP, submit dietary info, and leave feedback.

**Routes:** `/my-hub`, `/my-events`, `/my-events/[id]`, `/my-rewards`, `/hub/circles`
**Status:** DONE

## What's Here

- Client event list (upcoming/past)
- Per-event detail: booking progress card, countdown, chef contact, guest list with RSVP, dietary forms, pre-event info (parking, dress code, arrival), document sharing (recipe cards, wine pairings, photos, thank-you notes), photo consent, post-event feedback with ratings
- Dinner Circle card on live events: client-native invite copy, one-tap text-friendly join link, and host-side forwarding without leaving the event flow
- Dinner Circle Meals tab supports residency weekly planning, household dietary banners, meal conflict flags, attendance/comments, feedback, and chef-generated weekly shopping lists
- Loyalty rewards: points balance, tier progress, available rewards with redemption
- Account settings (preferences, notifications)
- "Browse Circles" button on `/my-hub` links to community circles discovery
- Clients can create, join, and participate in community circles
- Public join links keep inviter attribution so new circle joins can be traced back to the host or sender inside the thread
