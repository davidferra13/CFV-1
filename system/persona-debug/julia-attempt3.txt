# Persona Evaluation: Prep Kitchen Staff (Julia)

**Persona:** Julia, Prep Kitchen Staff
**Goal:** To efficiently execute prep lists, maintain accurate inventory, and ensure all ingredients are ready for service with minimal waste and confusion.
**Pain Points:** Inconsistent prep lists, difficulty tracking usage across multiple stations, manual logging of waste, and time wasted searching for misplaced items.
**Key Needs:** Digital, real-time inventory tracking; clear, actionable prep lists linked to recipes; simple waste logging.

***

## Evaluation Against System Capabilities

**1. Inventory Management:**
*   **System Strength:** Good. Tracks stock levels and allows for basic depletion logging.
*   **Gap:** Lacks granular, real-time tracking linked to specific prep tasks. It's good for *what* is low, but not *why* it's low or *where* it was used.

**2. Recipe Management & Prep Lists:**
*   **System Strength:** Moderate. Can generate lists based on recipes.
*   **Gap:** The prep list is static. It doesn't dynamically adjust based on actual usage or waste logged during the prep process.

**3. Workflow & Task Management:**
*   **System Strength:** Moderate. Can assign tasks.
*   **Gap:** Lacks the "station-based" workflow. Tasks are assigned generally, not tied to a physical workflow (e.g., "Prep Station A: Chop all onions for Soup X").

**4. Waste Tracking:**
*   **System Strength:** Weak. Requires manual entry, which is prone to error and forgetfulness.
*   **Gap:** No integrated, quick-capture waste logging mechanism tied to the prep task itself.

***

## Persona-Specific Recommendations

**1. Implement "Prep Task Check-Off":**
*   When a prep list is generated, each item should be a distinct, checkable task.
*   When the task is marked complete, the system should prompt for a **Usage Confirmation** (e.g., "Used 5 lbs of carrots. Confirm?"). This links the prep task directly to the inventory depletion.

**2. Introduce "Waste Capture Widget":**
*   On the prep screen, add a persistent, one-click "Log Waste" widget.
*   This widget should allow users to select the ingredient, estimate the waste amount (e.g., "1/4 lb"), and optionally select a reason (e.g., "Spoiled," "Over-prepped," "Trim Waste"). This feeds directly into the inventory adjustment.

**3. Enhance Station/Area Zoning:**
*   Allow managers to define physical zones (e.g., "Butchery Station," "Vegetable Prep Area").
*   Prep lists should be filterable or viewable by zone, guiding the user through a logical physical workflow, reducing search time.

***

## Summary Scorecard

| Feature | Current Score (1-5) | Improvement Needed | Priority |
| :--- | :--- | :--- | :--- |
| Real-Time Inventory Depletion | 3/5 | Link depletion directly to prep tasks. | High |
| Dynamic Prep List Adjustment | 2/5 | Must account for logged waste/over-prep. | High |
| Waste Logging | 2/5 | Needs a quick, integrated capture mechanism. | High |
| Workflow Guidance | 3/5 | Needs physical zoning/station workflow. | Medium |
| Usability (Speed) | 4/5 | Keep the interface extremely simple for high-volume use. | Medium |

**Overall Recommendation:** The system is strong on *tracking* but weak on *workflow integration*. It needs to move from being a record-keeping tool to an active, guiding tool that manages the physical process of preparation.
